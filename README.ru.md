## Аннотация

Современные LLM-сервисы хранят «память» о пользователе на своих серверах, что создаёт риски утечки, цензуры и юридического давления.

Данная архитектура предлагает **альтернативу**:
- Память хранится в зашифрованном виде на **независимом VPS**.
- LLM-провайдер получает доступ к памяти **только на время сессии**.
- Доступ контролируется **голосовой биометрией** пользователя.
- Система включает **стоп-слово** (самоуничтожение) и **старт-слово** (экстренная передача данных).
- Опциональная **геопривязка** создаёт криптографическое алиби.

---

## 1. Проблема

| Риск | Описание |
| :--- | :--- |
| **Утечка данных** | Взлом сервера, инсайдеры. |
| **Юридическое давление** | Суд может истребовать историю диалогов. |
| **Отсутствие контроля** | Пользователь не может гарантированно удалить память. |
| **Нет алиби** | Невозможно доказать своё местонахождение в момент диалога. |

---

## 2. Ключевые принципы

1. **Приватность по дизайну** — LLM-провайдер не хранит историю.
2. **Разделение ответственности** — пользователь, VPS и LLM — три субъекта.
3. **Эфемерность** — расшифрованная память только на устройстве пользователя и в RAM LLM.
4. **Биометрический контроль** — доступ по живому голосу.
5. **Неотказуемость** — все действия подписываются криптографически.

---

## 3. Компоненты

| Компонент | Функции |
| :--- | :--- |
| **Клиент** (телефон, ПК, робот) | Биометрия, шифрование, ключи, стоп-слова, локальные функции (запись, SOS, заметки) |
| **VPS** (арендованный сервер) | Хранение `memory.enc`, API `GET/PUT/DELETE`, аудит |
| **LLM-сервер** (провайдер) | Дешифровка памяти в RAM, генерация ответов, уничтожение после сессии |

**Важно:** Без интернета ИИ не работает. Локальные функции (запись, SOS) сохраняются.

---

## 4. Протокол работы

**Инициализация (1 раз):**
1. Пользователь записывает эталон голоса.
2. Клиент генерирует ключ AES-256.
3. Клиент создаёт пустой `memory.enc` и отправляет на VPS.

**Сессия:**
1. Пользователь начинает разговор.
2. Клиент верифицирует голос (liveness + сравнение).
3. Клиент отправляет LLM-серверу запрос, ID файла и ключ.
4. LLM забирает `memory.enc`, дешифрует, подгружает в контекст.
5. Диалог идёт. LLM обновляет память в RAM.
6. По окончании LLM шифрует обновлённую память, отправляет на VPS (`PUT`), затем **удаляет ключ и данные**.

## 5. Механизмы безопасности

### 5.1. Шифрование
- AES-256-GCM / ChaCha20-Poly1305.
- Ключ на устройстве, не передаётся (кроме LLM на время сессии).

### 5.2. Голосовая биометрия
- Вектор голоса (ECAPA-TDNN).
- Liveness detection (случайная фраза, анализ спектрограммы).

### 5.3. Стоп-слово (самоуничтожение)
- Клиент отправляет `DELETE /memory.enc` на VPS.
- VPS физически удаляет файл.

### 5.4. Старт-слово (Guardian Trigger)
- Защита от принуждения.
- Клиент расшифровывает память, шифрует публичными ключами получателей, отправляет по защищённым каналам. Исходный файл не удаляется.

### 5.5. Геопривязка (опционально)
- Клиент подписывает сессию блоком `timestamp + location + voice_hash + подпись`.
- Velocity Check (скорость > порога → предупреждение / отказ).

## 6. Несколько клиентов, синхронизация и безопасность

Пользователь может иметь **несколько клиентов** (телефон, часы, колонка).

### 6.1. Синхронизация
- Память на VPS.
- Локальные кэши клиентов не хранят полную историю.
- Изменения с одного клиента синхронизируются через VPS.

### 6.2. Блокировка потеряшки
- Со своего основного клиента или веб-интерфейса пользователь может:
  - **Заблокировать** утерянный клиент.
  - **Стереть** его локальные данные.
  - **Отвязать** от VPS.
- Требуется подтверждение паролем/биометрией.

---

## 7. Два режима приватности (выбор пользователя)

### 7.1. Режим «Абсолютная приватность» (ключ умирает с владельцем)
- Ключ только на устройстве, нигде не копируется.
- После смерти память **навсегда недоступна** (даже для следствия).
- **Никаких исключений.**

### 7.2. Режим «Наследуемая память» (с доступом по суду)
- Ключ депонируется у третьих лиц (нотариус, госбаза).
- Используется схема разделения ключа (Shamir).
- После смерти:
  - По решению суда — доступ для следствия (только чтение).
  - По завещанию — наследники получают доступ к наследуемому слою.
- **Личный слой уничтожается.**

Выбор режима делается **при жизни** и необратим (для Режима 1).

## 8. Ограничения

| Ограничение | Пояснение |
| :--- | :--- |
| **Задержки** | +1-2 сек на запрос к VPS |
| **Стоимость** | Аренда VPS дороже обычного чата |
| **Голосовая биометрия** | Не 100% защита от deepfake |
| **Юрисдикция VPS** | Данные могут быть изъяты (но зашифрованы) |
| **Взлом устройства** | Полный контроль над устройством ломает защиту |

## 9. Связь с другими концепциями

- **`SELF_IMPROVEMENT`** — системная инициатива (ИИ предлагает улучшения разработчикам).
- **`PROACTIVE_COMPANION`** — проактивный помощник (редкие предложения пользователю).
- **`CHILD_AI`** — детский режим, взросление, защита от травли.
- **`GRIEF_MODE`** — траурный режим и наследование памяти.
